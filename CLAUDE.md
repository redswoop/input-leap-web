## What This Is

Web UI for configuring [InputLeap](https://github.com/input-leap/input-leap) KVM servers. Replaces the Qt GUI with a browser-based topology editor that supports features the stock GUI can't (split-edge links, range-based edge intervals). Runs on the server machine, accessible from any LAN client.

## Commands

```bash
npm install          # installs both root and client deps (postinstall runs cd client && npm install)
npm run dev          # runs backend (Express :24802) + frontend (Vite :5173) concurrently with hot reload
npm run build        # builds Vue client into public/ for production
npm start            # runs production server (serves built frontend from public/)
```

Dev mode: open http://localhost:5173 (Vite proxies `/api` and `/ws` to :24802).
Production: open http://localhost:24802.

No test framework is configured.

## Architecture

```
Browser ‚Üê‚îÄ HTTP/WS ‚îÄ‚Üí Express+WS server (:24802) ‚Üê‚îÄ spawns ‚îÄ‚Üí input-leaps binary
                              ‚îÇ
                     input-leap.conf (INI-like)
                     layout.json (canvas positions)
```

**Two data files on disk:**
- `input-leap.conf` ‚Äî InputLeap's native config format (screens, links, aliases, options sections). Parsed/generated by `lib/config.js`.
- `layout.json` ‚Äî canvas positions/sizes for the topology editor (not part of InputLeap's format). Saved separately because InputLeap config has no concept of visual layout.

**Backend** (`server.js`, `lib/`): Express + WS. CommonJS (`"type": "commonjs"`).
- `lib/config.js` ‚Äî parser and generator for InputLeap's config file format. Handles range-based edge syntax like `down(0,50) = laptop`.
- `lib/process.js` ‚Äî manages spawning/stopping the `input-leaps` binary. Emits `log` and `status` events. Auto-discovers the binary across platforms.
- `server.js` ‚Äî REST API (`/api/config`, `/api/layout`, `/api/hostname`, `/api/status`, `/api/server/*`) + WebSocket (`/ws`) for live log/event streaming. Parses log lines into structured events (client connect/disconnect, screen switch).

**Frontend** (`client/`): Vue 3 SPA + Vite. ES modules.
- `composables/useConfig.js` ‚Äî singleton reactive store. Holds all app state (screens, links, aliases, options). Generates config text, handles save/load, polls InputLeap status. Shared across all components via the composable pattern (module-level refs, not per-instance).
- `composables/useWebSocket.js` ‚Äî singleton WebSocket connection with auto-reconnect. Provides reactive `logs`, `serverStatus`, and `events` refs.
- `lib/edge-detection.js` ‚Äî computes adjacency links from screen canvas positions. Uses a snap threshold (14px) to detect shared edges and generates range-based link objects with percentage intervals.
- `components/LayoutCanvas.vue` ‚Äî HTML5 Canvas topology editor with drag-and-drop, resize handles, snap-to-edge.
- `components/ScreenSettings.vue` ‚Äî per-screen config (modifier keys, aliases, OS-specific presets).
- `components/ServerTab.vue` ‚Äî server start/stop/restart controls, process options.
- `components/StatusTab.vue` ‚Äî live connection status and parsed events.
- `components/LogViewer.vue` ‚Äî collapsible live log panel at bottom of app.

**Key data flow:** Canvas drag ‚Üí `useConfig.screens` updated ‚Üí `edge-detection.detectEdges()` recomputes links ‚Üí `useConfig.configText` (computed) regenerates the config file text ‚Üí save sends both config + layout to backend.

## Config Format Notes

InputLeap config uses `section: <name>` / `end` blocks. Directions in config use `up`/`down` but internally normalized to `top`/`bottom` (see `normalizeDir`/`dirToConfig` in `lib/config.js`). Link ranges are percentages 0‚Äì100 along an edge.
